0. Key Clarifications From `immich-go` Documentation

The uploaded documentation clarifies several important points that affect how the GUI should generate commands.

0.1. Global command syntax is explicit

Documentation states:

immich-go [global-options] command sub-command [command-options] [path]

This means global options such as:

--log-level=DEBUG
--log-file=...
--log-type=JSON

should appear before the command:

immich-go --log-level=DEBUG upload from-folder ...

The current GUI builds:

return cmd + global_opts + cmd_opts + path_opt

which can generate:

immich-go upload from-folder --log-level=DEBUG ...

That is likely incorrect for global flags.

0.2. upload and archive share the same source sub-commands

Documentation says upload and archive both support:

from-folder
from-google-photos
from-icloud
from-picasa
from-immich

The GUI currently supports:

upload from-folder
upload from-google-photos
upload from-immich
archive from-folder
archive from-immich
stack

Missing possible tabs/features:

upload from-icloud
upload from-picasa
archive from-google-photos
archive from-icloud
archive from-picasa

Also, because upload and archive share the same source logic, the current archive from-folder and archive from-immich pages are missing many filtering options that likely belong there.

0.3. Google Takeout supports split ZIPs and glob patterns

Documentation examples:

immich-go upload from-google-photos --server=... --api-key=... /path/to/takeout-*.zip

The GUI currently allows only one path string and uses a folder browser.

This is insufficient for real Google Takeout imports, which often consist of many ZIP parts:

takeout-001.zip
takeout-002.zip
takeout-003.zip
...

The GUI should support:

  • Multi-file selection.
  • Drag-and-drop of multiple ZIP files.
  • Glob pattern input.
  • Optional automatic expansion of takeout-*.zip.

0.4. upload from-folder supports ZIP archives directly

Documentation example:

immich-go upload from-folder --server=... --api-key=... /path/to/photo-archive.zip

The GUI currently presents this as a folder-only workflow and uses:

QFileDialog.getExistingDirectory()

It should also allow selecting ZIP files.

0.5. immich-go supports environment variables for every flag

Documentation states:

IMMICH_GO_<COMMAND>[_<SUBCOMMAND>]_<OPTION_NAME>

Examples:

IMMICH_GO_UPLOAD_SERVER
IMMICH_GO_UPLOAD_API_KEY
IMMICH_GO_UPLOAD_FROM_IMMICH_FROM_SERVER
IMMICH_GO_UPLOAD_FROM_IMMICH_FROM_API_KEY
IMMICH_GO_ARCHIVE_WRITE_TO_FOLDER
IMMICH_GO_STACK_MANAGE_BURST

This is extremely useful for avoiding secrets on the command line.

The GUI should prefer passing API keys through environment variables where possible, especially if using an embedded QProcess runner.

1. Critical `immich-go` Command Builder Issues

1.1. Global flags are placed after the sub-command

Observation

Current code:

return cmd + global_opts + cmd_opts + path_opt

Example generated command:

immich-go upload from-folder --log-level=DEBUG --server=... --api-key=...

Documentation says global options should come before the command:

immich-go --log-level=DEBUG upload from-folder --server=... --api-key=...

Fix

Change command composition to:

return global_opts + cmd + cmd_opts + path_opt

Example:

def build_command(self, dry_run: bool) -> list[str]:
    ...
    return global_opts + cmd + cmd_opts + path_opt

Add tests:

def test_global_log_level_before_command():
    cmd = builder.build_upload_folder(
        state=state,
        server="http://localhost:2283",
        api_key="secret",
        log_level="DEBUG",
        dry_run=False,
    )

    assert cmd[0] == "--log-level=DEBUG"
    assert cmd[1] == "upload"
    assert cmd[2] == "from-folder"

1.8. `--concurrent-tasks` default should probably match CPU count

Observation

GUI default:

self.concurrent_tasks_spin.setValue(2)

Documentation says default is CPU cores, while config example shows 12.

If the GUI always emits --concurrent-tasks=2, it may unintentionally reduce performance.

Current code only emits when value is not 2:

conc = self.inputs["config"]["concurrent"].value()
if conc != 2:
    cmd_opts.append(f"--concurrent-tasks={conc}")

This avoids overriding when default is 2, but the default itself may be misleading.

Fix

Initialize spinbox to CPU count:

import os

cpu_count = os.cpu_count() or 2
self.concurrent_tasks_spin.setValue(min(max(cpu_count, 1), 20))

Then decide whether to emit only when changed from detected default.

Better:

self.default_concurrent_tasks = min(max(os.cpu_count() or 2, 1), 20)
self.concurrent_tasks_spin.setValue(self.default_concurrent_tasks)

Command builder:

conc = self.inputs["config"]["concurrent"].value()

if conc != self.default_concurrent_tasks:
    cmd_opts.append(f"--concurrent-tasks={conc}")

def browse_takeout_source(self):
    files, _ = QFileDialog.getOpenFileNames(
        self,
        "Select Takeout ZIP parts",

Use a QPlainTextEdit instead of QLineEdit for Takeout path input so users can paste multiple paths.

Command builder should support multiple paths:

def collect_paths(self, raw_text: str) -> list[str]:
    paths = []

    for line in raw_text.splitlines():
        line = line.strip()
        if not line:
            continue

        expanded = glob.glob(line, recursive=True)

        if expanded:
            paths.extend(expanded)
        else:
            paths.append(line)

    return paths

Then:

if tab_key == "upload-gp":
    path_opt.extend(self.collect_paths(c["path"].toPlainText()))

For other tabs, maybe keep single path.

3.2. Drag-and-drop should accept multiple files for Takeout

Observation

Current drop handler only uses first path:

if paths:
    target.setText(paths[0])

Fix

For Takeout, accept all paths:

def dropEvent(self, event: QDropEvent, target=None):
    if target is None:
        return

    paths = [url.toLocalFile() for url in event.mimeData().urls()]

    if not paths:
        return

    if isinstance(target, QPlainTextEdit):
        target.setPlainText("\n".join(paths))
    elif isinstance(target, QLineEdit):
        target.setText(paths[0])

    event.acceptProposedAction()

4. Security and Secret Handling

4.1. API keys are exposed in command preview and process arguments

Observation

Current confirm dialog shows:

--api-key=SECRET

This is risky.

Fix

At minimum, mask secrets in the UI:

SECRET_FLAGS = [
    "--api-key",
    "--from-api-key",
]

def mask_command_for_display(command_parts: list[str]) -> list[str]:
    masked = []

    for part in command_parts:
        hidden = False

        for flag in SECRET_FLAGS:
            prefix = f"{flag}="
            if part.startswith(prefix):
                masked.append(f"{flag}=********")
                hidden = True
                break

        if not hidden:
            masked.append(part)

    return masked

Use:

masked_parts = mask_command_for_display(full_cmd)
cmd_block.setPlainText(shlex.join(masked_parts))
4.2. Prefer environment variables for API keys

Documentation confirms environment variables exist.

Fix

Create a secret-through-environment mode.

Example:

def build_environment(self, tab_key: str) -> dict:
    env = os.environ.copy()

    server = self.inputs["config"]["server"].text().strip()
    api_key = self.inputs["config"]["api_key"].text().strip()

def verify_sha256(path: Path, expected: str) -> bool:

class DownloadThread(QThread):

        super().__init__()

6.2. Consider embedded `QProcess` runner

Observation

Current design opens an external terminal and then scans processes with psutil.

Problems:

  • Sandbox incompatibility.
  • No output capture.
  • No cancellation.
  • No exact exit code.
  • No completion notification.

Fix

Add optional embedded runner:

class ProcessRunner(QObject):
    output = Signal(str)
    finished = Signal(int)

    def __init__(self, parent=None):
        super().__init__(parent)
        self.process = QProcess(self)
        self.process.setProcessChannelMode(QProcess.ProcessChannelMode.MergedChannels)
        self.process.readyReadStandardOutput.connect(self._read_output)
        self.process.finished.connect(self._finished)

    def _read_output(self):
        data = self.process.readAllStandardOutput()
        self.output.emit(bytes(data).decode("utf-8", errors="replace"))

    def _finished(self, exit_code, exit_status):
        self.finished.emit(exit_code)

    def start(self, program: str, args: list[str], env: dict):
        self.process.start(program, args, env=env)

    def cancel(self):
        self.process.kill()

14. Qt Web-Search Verification Checklist

Since I cannot perform a live web search here, you can verify the Qt-specific conclusions using these searches.

14.1. Custom toggle button

Search:

PySide6 custom toggle button QAbstractButton
Qt custom switch widget accessibility
QAbstractButton keyPressEvent space toggle

What to confirm:

  • Custom buttons should usually inherit QAbstractButton.
  • Buttons should support keyboard activation.
  • Accessible names should be set.
14.2. Dark mode detection

Search:

Qt QStyleHints colorScheme
PySide6 Qt.ColorScheme.Dark
Qt colorSchemeChanged signal

What to confirm:

  • QGuiApplication.styleHints().colorScheme() is appropriate in modern Qt.
  • Guarding with hasattr is good for compatibility.
14.3. QThread termination

Search:

Qt QThread terminate discouraged
PySide6 QThread cancel gracefully
QThread cooperative cancellation

What to confirm:

  • QThread.terminate() is unsafe.
  • Cooperative cancellation is recommended.
14.4. HiDPI SVG icons

Search:

PySide6 QSvgRenderer QPixmap devicePixelRatio
Qt HiDPI QIcon QPixmap setDevicePixelRatio

What to confirm:

  • Fixed 20x20 pixmaps may blur on HiDPI.
  • setDevicePixelRatio() or multi-size QIcon is preferred.
14.5. QPushButton checked icon color

Search:

QPushButton checked icon color QSS
Qt QIcon selected state QPushButton checked
recolor SVG icon QPushButton checked

What to confirm:

  • QSS can change text color but not easily recolor QIcon.
  • Custom painting or icon swapping is often needed.
14.6. QSettings vs TOML

Search:

PySide6 QSettings cross platform
Python platformdirs TOML config
Qt application organization name

What to confirm:

  • QSettings is functional but platform-specific.
  • TOML + platformdirs gives more predictable config files.
  • Secrets should not be stored in plain config.

15. Final Summary

The biggest documentation-driven corrections are:

  1. Global flags must come before the command.
  2. upload and archive share many source options, so archive pages are incomplete.
  3. Google Takeout needs multi-ZIP/glob support.
  4. upload from-folder should support ZIP files.
  5. Environment variables can be used to avoid exposing API keys.
  6. Several collected options are unused: client_timeout, device_uuid.
  7. Some flags should be scoped carefully: --pause-immich-jobs, --on-errors, --api-trace.
  8. Missing source tabs: iCloud, Picasa, archive Google Photos.

The biggest Qt-driven corrections are:

  1. Possible syntax/formatting defects must be verified.
  2. SwitchButton should be a proper QAbstractButton.
  3. Icons need HiDPI, caching, and checked-state coloring.
  4. Network and downloads must be fully asynchronous and cancellable.
  5. Process execution should move toward QProcess with --no-ui.
  6. Configuration should move to TOML + keychain.
  7. Tests should target a decoupled command builder, not GUI internals.
